# snap-meal v0.1.3

发布日期：2026-06-25

本版本重点完善微信小程序用户端体验，使小程序点餐页面的呈现效果与 Web 用户点餐端保持一致，同时补强管理端菜品和分类维护能力。

## 更新重点

### 微信小程序端体验统一

- 小程序点餐首页视觉风格调整为与 Web 用户点餐端一致：
  - 深绿色头部区域
  - 米白色页面背景
  - 硬边界菜品卡片
  - 青柠色圆形加购按钮
  - 深色底部购物车栏

- 小程序首页新增底部购物车弹层：
  - 点击底部购物车区域后，不再跳转独立购物车页面
  - 当前页面向上弹出购物车明细面板
  - 交互方式与 Web 用户端购物车面板保持一致

- 小程序购物车弹层支持完整购物车管理：
  - 查看已选菜品
  - 查看单价、数量、小计和合计
  - 支持 `+` / `-` 调整数量
  - 数量减到 0 时自动移除菜品
  - 支持清空购物车
  - 支持直接进入结算流程

- 移除小程序冗余购物车页面和底部 TabBar：
  - 购物车功能集中到首页弹层
  - 订单入口保留在首页头部
  - 点餐流程更接近 Web 用户端

- 小程序补充轻量动态效果：
  - 菜品卡片上浮淡入
  - 加购按钮短暂从 `+` 变为 `✓`
  - 购物车面板向上弹出

### 管理端能力完善

- 菜品管理新增页面内表单：
  - 新增菜品时可选择分类
  - 编辑菜品时可调整名称、分类、价格和起售状态

- 分类管理逻辑简化：
  - 商家侧只保留分类名称和启用状态
  - 分类类型和排序字段由后端兼容处理，不再作为商家操作项暴露

### 文档更新

- README 新增 v0.1.3 更新说明。
- README 补充小程序真机调试步骤。
- 明确说明真机调试时需要将 `miniprogram/app.js` 中的 `baseUrl` 从 `localhost` 临时改为电脑局域网 IP。
- 明确说明发布或提交前应将 `baseUrl` 改回 `http://localhost:8080`。

## 注意事项

真机调试时，手机无法通过 `localhost` 访问电脑后端。需要临时将：

```javascript
baseUrl: 'http://localhost:8080'
```

改为电脑局域网 IP，例如：

```javascript
baseUrl: 'http://192.168.1.23:8080'
```

调试完成并准备提交代码前，应改回：

```javascript
baseUrl: 'http://localhost:8080'
```
